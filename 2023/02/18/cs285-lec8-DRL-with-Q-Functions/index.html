Replay Buffer

In online q-learning we have:

  1. take an action aia_iai​ and observe (si,ai,si′,ri)(s_i, a_i, s'_i, r_i)(si​,ai​,si′​,ri​)
  2. yi=ri+γmax⁡a′Qϕ(si′,ai′)y_i = r_i + \gamma \max_{a'}Q_\phi(s'_i, a'_i)yi​=ri​+γmaxa′​Qϕ​(si′​,ai′​)
  3. ϕ←ϕ−α∂Qϕ(si,ai)∂ϕ(Qϕ(si,ai)−yi)\phi \leftarrow \phi - \alpha \frac{\partial Q_\phi(s_i, a_i)}{\partial \phi}(Q_\phi(s_i, a_i) - y_i)ϕ←ϕ−α∂ϕ∂Qϕ​(si​,ai​)​(Qϕ​(si​,ai​)−yi​)

to combine step2 and step3, we get:

  1. take an action aia_iai​ and observe (si,ai,si′,ri)(s_i, a_i, s'_i, r_i)(si​,ai​,si′​,ri​)
  2. ϕ←ϕ−α∂Qϕ(si,ai)∂ϕ(Qϕ(si,ai)−[ri+γmax⁡a′Qϕ(si′,ai′)])⏟target value\phi \leftarrow \phi - \alpha \frac{\partial Q_\phi(s_i, a_i)}{\partial \phi}(Q_\phi(s_i, a_i) - \underbrace{[r_i + \gamma \max_{a'}Q_\phi(s'_i, a'_i)])}_{\text{target value}}ϕ←ϕ−α∂ϕ∂Qϕ​(si​,ai​)​(Qϕ​(si​,ai​)−target value[ri​+γa′max​Qϕ​(si′​,ai′​)])​​

Because all the samples we collected are happened sequentially, they are similar and strongly correlated, and the target value is always changing.

synchronized/asynchronized parallel Q-learning
To solve this problem, we could use synchronized parallel Q-learning or asynchronized parallel Q-learning as used in actor-critic:

replay buffer
Use replay buffers to generate data samples, we now have:

  1. sample a batch (si,ai,si′,ri)(s_i, a_i, s'_i, r_i)(si​,ai​,si′​,ri​) from B\mathcal{B}B
  2. ϕ←ϕ−α∑i∂Qϕ(si,ai)∂ϕ(Qϕ(si,ai)−[ri+γmax⁡a′Qϕ(si′,ai′)])\phi \leftarrow \phi - \alpha \sum_i\frac{\partial Q_\phi(s_i, a_i)}{\partial \phi}(Q_\phi(s_i, a_i) - [r_i + \gamma \max_{a'}Q_\phi(s'_i, a'_i)])ϕ←ϕ−α∑i​∂ϕ∂Qϕ​(si​,ai​)​(Qϕ​(si​,ai​)−[ri​+γmaxa′​Qϕ​(si′​,ai′​)])

This way samples are no longer correlated, and due to multiple samples in one batch, we have low-variance.

Target Network & Predict Network

In q-learning, when we need to update ϕ\phiϕ, we’ll need to calculate:

ϕ←ϕ−α∑i∂Qϕ(si,ai)∂ϕ(Qϕ(si,ai)−[ri+γmax⁡a′Qϕ(si′,ai′)]⏟no gradient through target value)\phi \leftarrow \phi - \alpha \sum_i\frac{\partial Q_\phi(s_i, a_i)}{\partial \phi}(Q_\phi(s_i, a_i) - \underbrace{[r_i + \gamma \max_{a'}Q_\phi(s'_i, a'_i)]}_{\text{no gradient through target value}}) ϕ←ϕ−αi∑​∂ϕ∂Qϕ​(si​,ai​)​(Qϕ​(si​,ai​)−no gradient through target value[ri​+γa′max​Qϕ​(si′​,ai′​)]​​)

The target value every step, so the network is hard to converage, we’ll need to use another less-changed network.

If we take K=1K=1K=1 then we’ll have DQN:

  1. take some action aia_iai​ and observe (si,ai,si′,ri)(s_i, a_i, s'_i, r_i)(si​,ai​,si′​,ri​), add it to B\mathcal{B}B, the actions were generated by ϕ\phiϕ and we might use epsilon-greedy here.
  2. sample mini-batch (sj,aj,sj′,rj)(s_j, a_j, s'_j, r_j)(sj​,aj​,sj′​,rj​) from B\mathcal{B}B uniformly
  3. compute yj=rj+γmax⁡aj′Qϕ′(sj′,aj′)y_j = r_j + \gamma \max_{a'_j}Q_{\phi'}(s'_{j}, a'_j)yj​=rj​+γmaxaj′​​Qϕ′​(sj′​,aj′​) using target network Qϕ′Q_{\phi'}Qϕ′​
  4. ϕ←ϕ−α∑j∂Qϕ(sj,aj)∂ϕ(Qϕ(sj,aj)−yj)\phi \leftarrow \phi - \alpha \sum_j\frac{\partial Q_\phi(s_j, a_j)}{\partial \phi}(Q_\phi(s_j, a_j) - y_j)ϕ←ϕ−α∑j​∂ϕ∂Qϕ​(sj​,aj​)​(Qϕ​(sj​,aj​)−yj​)
  5. update ϕ′\phi'ϕ′: copy ϕ\phiϕ every N steps

This way would lead the step right after the update of ϕ′\phi'ϕ′ to be only 1 step older, and the step right before to be the oldest. We could use Polyak Averaging (an optimization technique that sets final parameters to an average of recent parameters visited in the optimization trajectory) to be fair to every step.

When update ϕ′\phi'ϕ′, now we do:

ϕ′←τϕ′+(1−τ)ϕ\phi' \leftarrow \tau \phi' + (1 - \tau) \phi ϕ′←τϕ′+(1−τ)ϕ

Improve Q-learning

Problem: Overestimate the Q Value

Q-learning tends to overestimate the q value, because we calculate yjy_jyj​ as follows:

yj=rj+γmax⁡aj′Qϕ′(sj′,aj′)⏟here’s the problemy_j = r_j + \gamma \underbrace{\max_{a'_j}Q_{\phi'}(s'_{j}, a'_j)}_{\text{here's the problem}} yj​=rj​+γhere’s the problemaj′​max​Qϕ′​(sj′​,aj′​)​​

Because E[max⁡(X1,X2)]≥max⁡(E[X1],E[X2])E[\max(X_1, X_2)] \geq \max(E[X_1], E[X_2])E[max(X1​,X2​)]≥max(E[X1​],E[X2​]), and Q←r+γEQ \leftarrow r + \gamma EQ←r+γE, the q-learning will overestimate the true q value.

Solution: Double Q-learning

Note that we select action a′a'a′ from Qϕ′(s′,a′)Q_{\phi'}(s', a')Qϕ′​(s′,a′), and we also estimate q-value using it, so:

max⁡a′Qϕ′(s′,a′)=max⁡a′Qϕ′⏟value also from Qϕ′(s′,arg max⁡a′Qϕ′(s′,a′)⏟action selected from Qϕ′)\max_{a'}Q_{\phi'}(s', a') = \max_{a'}\underbrace{Q_{\phi'}}_{\text{value also from }Q_{\phi'}}(s', \underbrace{\argmax_{a'}Q_{\phi'}(s',a')}_{\text{action selected from }Q_{\phi'}}) a′max​Qϕ′​(s′,a′)=a′max​value also from Qϕ′​Qϕ′​​​(s′,action selected from Qϕ′​a′argmax​Qϕ′​(s′,a′)​​)

If Qϕ′(s′,a′)Q_{\phi'}(s', a')Qϕ′​(s′,a′) has noise, it would be multiplied. Thus we use two separate networks to choose the action and evaluate value.

QϕA(s′,a′)←r+γQϕB(s′,arg max⁡a′QϕA(s′,a′))QϕB(s′,a′)←r+γQϕA(s′,arg max⁡a′QϕB(s′,a′))\begin{aligned} Q_{\phi_A}(s', a') &\leftarrow r + \gamma Q_{\phi_B}(s', \argmax_{a'} Q_{\phi_A}(s', a')) \\ Q_{\phi_B}(s', a') &\leftarrow r + \gamma Q_{\phi_A}(s', \argmax_{a'} Q_{\phi_B}(s', a')) \end{aligned}QϕA​​(s′,a′)QϕB​​(s′,a′)​←r+γQϕB​​(s′,a′argmax​QϕA​​(s′,a′))←r+γQϕA​​(s′,a′argmax​QϕB​​(s′,a′))​

In practice, we already have target network and predict network, so we will use them as ϕA,ϕB\phi_A, \phi_BϕA​,ϕB​

So a double q-learning will be:

y=r+γmax⁡a′Qϕ′(s′,arg max⁡a′Qϕ(s′,a′))y = r + \gamma \max_{a'}Q_{\phi'}(s', \argmax_{a'}Q_\phi(s', a')) y=r+γa′max​Qϕ′​(s′,a′argmax​Qϕ​(s′,a′))

Note we use predict network QϕQ_\phiQϕ​ to choose actions, use target network Qϕ′Q_{\phi'}Qϕ′​ to evaluate the value.

Overall this double-q network helps a lot in practice, and it has no downsides.

Multiple Steps

Since we calculate q-learning target by:

yj,t=rj,t+γmax⁡aj,t+1Qϕ′(sj,t+1,aj,t+1)y_{j,t} = r_{j,t} + \gamma \max_{a_{j, t+1}}Q_{\phi'}(s_{j,t+1}, a_{j, t+1}) yj,t​=rj,t​+γaj,t+1​max​Qϕ′​(sj,t+1​,aj,t+1​)

The rj,tr_{j,t}rj,t​ is the only values that matter if Qϕ′Q_{\phi'}Qϕ′​ is bad, and Qϕ′Q_{\phi'}Qϕ′​ is the only values that matter if Qϕ′Q_{\phi'}Qϕ′​ is good. So if our Qϕ′Q_{\phi'}Qϕ′​ is bad at first, we may get stuck. If we calculate rewards at multiple steps, we could avoid this problem.

Like in actor-critic, we now have:

yj,t=∑t′=tt+N−1γt−t′rj,t′+γNmax⁡aj,t+NQϕ′(sj,t+N,aj,t+N)y_{j,t} = \sum_{t'=t}^{t+N-1} \gamma^{t-t'}r_{j,t'} + \gamma^N \max_{a_{j, t+N}}Q_{\phi'}(s_{j,t+N}, a_{j, t+N}) yj,t​=t′=t∑t+N−1​γt−t′rj,t′​+γNaj,t+N​max​Qϕ′​(sj,t+N​,aj,t+N​)

But this only works on on-policy.

In practice, just ignore this restriction and apply it to off-policy, it still works well somehow…

Countinuous Actions

To find the arg max⁡a′Qϕ′(s′,a′)\argmax_{a'}Q_{\phi'}(s', a')a′argmax​Qϕ′​(s′,a′) is easy when actions are discreate, but when actions are continuous, it would take much time.

Approximate

We could approximate the arg max⁡a′Qϕ′(s′,a′)\argmax_{a'}Q_{\phi'}(s', a')a′argmax​Qϕ′​(s′,a′) by using monte-carlo like methods:

max⁡aQ(s,a)≈max⁡{Q(s,a1),Q(s,a2),…Q(s,an)}\max_a Q(s, a) \approx \max \{ Q(s, a_1), Q(s, a_2), \dots Q(s, a_n)\} amax​Q(s,a)≈max{Q(s,a1​),Q(s,a2​),…Q(s,an​)}

(a1,a2,…,aN)(a_1, a_2, \dots, a_N)(a1​,a2​,…,aN​) are sampled from some distribution

Works well for about 40 dimensions.

Choose Easily Maximizable Q-functions

At the cost of reducing the representation capacity of q-function, we could make arg max⁡aQ\argmax_aQaargmax​Q and max⁡aQ\max_aQmaxa​Q very easy, for example we could use NAF/Normalized Advantage Functions

Qϕ(s,a)=−12(a−μϕ(s))TPϕ(s)(a−μϕ(s))+Vϕ(s)Q_\phi(s,a) = -\frac{1}{2}(a - \mu_\phi(s))^TP_\phi(s)(a - \mu_\phi(s))+V_\phi(s) Qϕ​(s,a)=−21​(a−μϕ​(s))TPϕ​(s)(a−μϕ​(s))+Vϕ​(s)

and we get:

arg max⁡aQϕ(s,a)=μϕ(s)max⁡aQϕ(s,a)=Vϕ(s)\begin{aligned} \argmax_a Q_\phi(s,a) &= \mu_\phi(s) \\ \max_aQ_\phi(s,a) &= V_\phi(s) \end{aligned}aargmax​Qϕ​(s,a)amax​Qϕ​(s,a)​=μϕ​(s)=Vϕ​(s)​
