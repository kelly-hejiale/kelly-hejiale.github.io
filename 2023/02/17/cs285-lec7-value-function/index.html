cs285-lec7-value function

Edited on 2023-09-30 In technical , course notes Views:
If we don't want to learn a policy, we could learn value funtions.

Summary

We don’t need to learn a policy, instead we’ll use the policy that choose the action to get maximum Q-value at the current state.

For example, offline Q-learning:

online Q-learning:

  1. take an action aia_iai​ and observe (si,ai,si′,ri)(s_i, a_i, s'_i, r_i)(si​,ai​,si′​,ri​)
  2. yi=ri+γmax⁡a′Qϕ(si′,ai′)y_i = r_i + \gamma \max_{a'}Q_\phi(s'_i, a'_i)yi​=ri​+γmaxa′​Qϕ​(si′​,ai′​)
  3. ϕ←ϕ−α∂Qϕ(si,ai)∂ϕ(Qϕ(si,ai)−yi)\phi \leftarrow \phi - \alpha \frac{\partial Q_\phi(s_i, a_i)}{\partial \phi}(Q_\phi(s_i, a_i) - y_i)ϕ←ϕ−α∂ϕ∂Qϕ​(si​,ai​)​(Qϕ​(si​,ai​)−yi​)

This method is not guaranteed to converage.

Tabular reinforcement learning

Assume we know the transition probability p(s′∣s,a)p(s'|s,a)p(s′∣s,a), and st,ats_t, a_tst​,at​ are both discreate. Then we could do tabular reinforcement learning, store Vπ(st)V^\pi(s_t)Vπ(st​) into a table, update Vπ(st)V^\pi(s_t)Vπ(st​) at every one step by:

Vπ(s)←r(s,π(s))+γEs′∼p(s′∣s,π(s))[Vπ(s′)]V^\pi(s) \leftarrow r(s, \pi(s)) + \gamma E_{s' \sim p(s'|s, \pi(s))}[V^\pi(s')] Vπ(s)←r(s,π(s))+γEs′∼p(s′∣s,π(s))​[Vπ(s′)]

In order to simplify the above equation, we could use QQQ to replace VVV, so the procedure will be:

  1. construct the q-value table by: Q(s,a)=r(s,a)+γE[V(s′)]Q(s,a) = r(s,a) + \gamma E[V(s')]Q(s,a)=r(s,a)+γE[V(s′)]
  2. V(s)←max⁡aQ(s,a)V(s) \leftarrow \max_a Q(s,a)V(s)←maxa​Q(s,a)

Notice we would use the q-value to update value function at step2, so just combine them together, we would get rid of value function entirely, now the procedure will be:

  1. construct the q-value table by: Q(s,a)=r(s,a)+γEQ(s,a) = r(s,a) + \gamma EQ(s,a)=r(s,a)+γE, EEE is the expected value of the next time step

Fitted Q-iteration

When the number of states is large, it’s impossible to construct a table, so we use neural network here.

We would train a neural network that maps state to an estimated reward: V(s)→RV(s) \rightarrow \mathcal{R}V(s)→R, then we will optimize this network by:

L(ϕ)=12∥Vϕ(s)−max⁡aQπ(s,a)∥2\mathcal{L}(\phi) = \frac{1}{2}\|V_\phi(s) - \max_aQ^\pi(s,a)\|^2 L(ϕ)=21​∥Vϕ​(s)−amax​Qπ(s,a)∥2

Note: sometimes the gradient might be too big, so we may change the loss function to Huber loss or use gradient clip.

The training will be:

  1. set yi←max⁡ai(r(si,ai)+γE[Vϕ(si′)])y_i \leftarrow \max_{a_i}(r(s_i, a_i) + \gamma E[V_\phi(s'_i)])yi​←maxai​​(r(si​,ai​)+γE[Vϕ​(si′​)])
  2. set ϕ←arg min⁡ϕ12∥Vϕ(s)−yi∥2\phi \leftarrow \argmin_\phi \frac{1}{2}\|V_\phi(s) - y_i\|^2ϕ←ϕargmin​21​∥Vϕ​(s)−yi​∥2

Notice we still need to know the transition probability to calculate Es′∼p(s′∣s,a)E_{s'\sim p(s'|s,a)}Es′∼p(s′∣s,a)​ in step1, to fit this, we approxiate E[Vϕ(si′)]≈max⁡a′Qϕ(si′,ai′)E[V_\phi(s'_i)] \approx \max_{a'}Q_\phi(s'_i, a'_i)E[Vϕ​(si′​)]≈maxa′​Qϕ​(si′​,ai′​)

This q-learning works for off-policy samples, and it has only one network, no need to do policy gradient. But it’s not guaranteed to converage.

Explore and Exploit

If the action we choose at very fisrt is bad, and we only take actions to maximize the Q value, we might stuck in the bad loop.
In order to solve this, we need to do some actions to explore.
epsilon-greedy
We take the actions that arg max Q value at the probability of 1−ϵ1-\epsilon1−ϵ, that is:

π′(at∣st)={1−ϵ,if    at=arg max⁡atAπ(st,at)ϵ/(∣A∣−1),otherwise\pi'(a_t|s_t) = \begin{cases} \begin{aligned} &1-\epsilon, & &\text{if} \;\; a_t = \argmax_{a_t}A^\pi(s_t, a_t) \\ &\epsilon / (|\mathcal{A}|-1), & &\text{otherwise} \end{aligned} \end{cases} π′(at​∣st​)=⎩⎨⎧​​1−ϵ,ϵ/(∣A∣−1),​​ifat​=at​argmax​Aπ(st​,at​)otherwise​​

So when the policy is good, we could use smaller ϵ\epsilonϵ and larger ϵ\epsilonϵ when the policy is bad.

Usually, starts with large epsilon and gradually decrease.
Boltzmann exploration
Select actions in proportion to some positive transformation of Q values, and the most popular one is exp. Using this the best actions would appear at the highest frequency.

π(at∣st)∝exp⁡(Qϕ(st,at))\pi(a_t|s_t) \propto \exp(Q_\phi(s_t, a_t)) π(at​∣st​)∝exp(Qϕ​(st​,at​))
