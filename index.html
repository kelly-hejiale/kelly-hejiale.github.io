Fitted Q-iteration

When the number of states is large, it’s impossible to construct a table, so we use neural network here.

We would train a neural network that maps state to an estimated reward: V(s)→RV(s) \rightarrow \mathcal{R}V(s)→R, then we will optimize this network by:

L(ϕ)=12∥Vϕ(s)−max⁡aQπ(s,a)∥2\mathcal{L}(\phi) = \frac{1}{2}\|V_\phi(s) - \max_aQ^\pi(s,a)\|^2 L(ϕ)=21​∥Vϕ​(s)−amax​Qπ(s,a)∥2

Explore and Exploit

If the action we choose at very fisrt is bad, and we only take actions to maximize the Q value, we might stuck in the bad loop.
In order to solve this, we need to do some actions to explore.
epsilon-greedy
We take the actions that arg max Q value at the probability of 1−ϵ1-\epsilon1−ϵ, that is:

π′(at∣st)={1−ϵ,if    at=arg max⁡atAπ(st,at)ϵ/(∣A∣−1),otherwise\pi'(a_t|s_t) = \begin{cases} \begin{aligned} &1-\epsilon, & &\text{if} \;\; a_t = \argmax_{a_t}A^\pi(s_t, a_t) \\ &\epsilon / (|\mathcal{A}|-1), & &\text{otherwise} \end{aligned} \end{cases} π′(at​∣st​)=⎩⎨⎧​​1−ϵ,ϵ/(∣A∣−1),​​ifat​=at​argmax​Aπ(st​,at​)otherwise​​

So when the policy is good, we could use smaller ϵ\epsilonϵ and larger ϵ\epsilonϵ when the policy is bad.

Boltzmann exploration
Select actions in proportion to some positive transformation of Q values, and the most popular one is exp. Using this the best actions would appear at the highest frequency.

π(at∣st)∝exp⁡(Qϕ(st,at))\pi(a_t|s_t) \propto \exp(Q_\phi(s_t, a_t)) π(at​∣st​)∝exp(Qϕ​(st​,at​))

Eligibility traces & n-steps returns

Actually we could see actor-critic as calculating A^π(si,ai)=r(si,ai)+γV^ϕπ(si′)−V^ϕπ(si)\hat{A}^\pi(s_i, a_i) = r(s_i, a_i) + \gamma \hat{V}^\pi_\phi(s'_i) - \hat{V}^\pi_\phi(s_i)A^π(si​,ai​)=r(si​,ai​)+γV^ϕπ​(si′​)−V^ϕπ​(si​) using rewards at current step, and policy gradient using rewards from all steps: A^π(si,ai)=∑t′=tTγt′−tr(si,t′,ai,t′)−V^ϕπ(si,t)\hat{A}^\pi(s_i, a_i)=\sum_{t' = t}^T \gamma^{t'-t}r(s_{i,t'}, a_{i, t'}) - \hat{V}^\pi_\phi(s_{i,t})A^π(si​,ai​)=∑t′=tT​γt′−tr(si,t′​,ai,t′​)−V^ϕπ​(si,t​).

As we mentioned above, actor-critic has lower variance but high bias (if the estimated value is wrong, which is very common), while policy gradient has no bias but high variance (because it estimates on every single sample).

So maybe we could find a point between these two, using rewards from some steps to calculate A^π(si,ai)\hat{A}^\pi(s_i, a_i)A^π(si​,ai​):

A^π(st,at)=∑t′=tt+nγt′−tr(st′,at′)−V^ϕπ(st)+γnV^ϕπ(st+n)\hat{A}^\pi(s_t, a_t) = \sum_{t' = t}^{t+n} \gamma^{t'-t}r(s_{t'}, a_{t'}) - \hat{V}^\pi_\phi(s_{t}) + \gamma^n \hat{V}^\pi_\phi(s_{t+n}) A^π(st​,at​)=t′=t∑t+n​γt′−tr(st′​,at′​)−V^ϕπ​(st​)+γnV^ϕπ​(st+n​)

Moreover, we could use attention-like mechanism to get a weighted combination of n-step returns:

A^GAEπ(st,at)=∑n=1∞wnA^nπ(st,at)=∑t′=t∞(γλ)t′−t(r(si,ai)+γV^ϕπ(si′)−V^ϕπ(si))⏟δt′\begin{aligned} \hat{A}^\pi_{GAE}(s_t, a_t) &= \sum_{n=1}^\infin w_n \hat{A}^\pi_n(s_t, a_t) \\ &= \sum_{t'=t}^\infin (\gamma \lambda)^{t'-t} \underbrace{\Big(r(s_i, a_i) + \gamma \hat{V}^\pi_\phi(s'_i) - \hat{V}^\pi_\phi(s_i)\Big)}_{\delta_{t'}} \end{aligned}A^GAEπ​(st​,at​)​=n=1∑∞​wn​A^nπ​(st​,at​)=t′=t∑∞​(γλ)t′−tδt′​(r(si​,ai​)+γV^ϕπ​(si′​)−V^ϕπ​(si​))​​​

The γλ\gamma \lambdaγλ here plays a similar role as discount factor, which will reduce variance.

bert的架构：multi-layer bidirectional Transformer encoder

使用了WordPiece作为embedding

对每个sentence pair，用SEP分开两个句子，然后对每个token增加一个embedding表示属于sentence A还是sentence B。
每个token的representation来自3个部分：token embedding, segment embedding, position embedding，即：

之前的LM都是单向的：要么从左向右，要么从右向左，理论上双向的模型应该比单向的模型效果好，但是双向的模型会直接“看到”要预测的目标。bert提出随机选择一部分token mask掉，然后预测这些mask掉的token

上面高亮的部分，正是ernie改进的点:ernie的mask不再是随机mask，而是选择句子中的entity进行mask

  2. 10%替换为随机token
  3. 10%保持不变

上面训LM的部分已经可以了，但是LM对句子之间的关系学习不好，因此还有专门对句子之间关系的模型。同样的训练数据，将sentence B改为：

  1. 50%情况下是sentence A的下一句，label = IsNext
  2. 50%情况下是语料库中的随机一句，label = NotNext

总结

在传统的预训练语言模型中引入知识图谱，对token, entity分别用bert进行embedding，然后利用information fusion更新embedding，最终以token到entity的条件概率交叉熵作为损失函数

变量定义

token序列：{w1,…,wn}\{w_1, \dots, w_n\}{w1​,…,wn​}，其中nnn为token长度
实体序列：{e1,…,em}\{e_1, \dots, e_m\}{e1​,…,em​}，其中mmm为实体序列的长度
包含全部token的集合V\mathcal{V}V
包含kg全部实体的集合E\mathcal{E}E
如果一个token w∈Vw \in \mathcal{V}w∈V有对应的e∈Ee \in \mathcal{E}e∈E，则定义对应关系为f(w)=ef(w) = ef(w)=e

模型架构

T-encoder的layer为N
K-encoder的layer为M

对输入的token序列{w1,…,wn}\{w_1, \dots, w_n\}{w1​,…,wn​}及其对应的实体序列{e1,…,em}\{e_1, \dots, e_m\}{e1​,…,em​}，t-encoder先把token embedding, segment embedding, positional embedding加起来，得到input embedding，然后计算feature:

{w1,…,wn}=T-encoder({w1,…,wn})\{w_1, \dots, w_n\} = \text{T-encoder(}\{w_1, \dots, w_n\}\text{)} {w1​,…,wn​}=T-encoder({w1​,…,wn​})

其中：T-encoder是一个multi-layer的双向transformer（即bert）

用TransE模型把实体转化为实体embedding {e1,…,em}\{e_1, \dots, e_m\}{e1​,…,em​}，然后输入K-encoder计算final output embedding:

{w1o,…,wno},{e1o,…,emo}=K-encoder({w1,…,wn},{e1,…,em})\{w_1^o, \dots, w_n^o\},\{e_1^o, \dots, e_m^o\} = \text{K-encoder(}\{w_1, \dots, w_n\}, \{e_1, \dots, e_m\}\text{)} {w1o​,…,wno​},{e1o​,…,emo​}=K-encoder({w1​,…,wn​},{e1​,…,em​})

K-encoder结构

k-encoder由多层aggregator叠起来
对每第i层aggregator，其输入为：{w1i−1,…,wni−1},{e1i−1,…,emi−1}\{w_1^{i-1}, \dots, w_n^{i-1}\},\{e_1^{i-1}, \dots, e_m^{i - 1}\}{w1i−1​,…,wni−1​},{e1i−1​,…,emi−1​}

{w1i,…,wni}=MH-ATTs({w1i−1,…,wni−1}){e1i,…,eni}=MH-ATTs({e1i−1,…,eni−1})\{w_1^{i}, \dots, w_n^{i}\} = \text{MH-ATTs}(\{w_1^{i-1}, \dots, w_n^{i-1}\}) \\ \{e_1^{i}, \dots, e_n^{i}\} = \text{MH-ATTs}(\{e_1^{i-1}, \dots, e_n^{i-1}\}) {w1i​,…,wni​}=MH-ATTs({w1i−1​,…,wni−1​}){e1i​,…,eni​}=MH-ATTs({e1i−1​,…,eni−1​})
