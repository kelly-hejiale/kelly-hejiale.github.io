Attention is all you need

发表于 2021-08-26 分类于 技术 ， ml/dl ， 论文

一直闻名transformer，今天才知道原来就是attention这篇文章……

总结

只使用attention，没有cnn/rnn结构
创新点

  1. 只有attention结构
  2. 多头注意力
  3. position encoding

细节

机器翻译的大多数模型，encoder-decoder架构
encoder: (x1,x2,…,xn)→(z1,z2,…,zn)(x_1, x_2, \dots, x_n) \to (z_1, z_2, \dots, z_n)(x1​,x2​,…,xn​)→(z1​,z2​,…,zn​)
decoder: (z1,z2,…,zn)→(y1,y2,…,ym)(z_1, z_2, \dots, z_n) \to (y_1, y_2, \dots, y_m)(z1​,z2​,…,zn​)→(y1​,y2​,…,ym​)

架构图：

encoder: Nx=6N_x = 6Nx​=6，每一个layer中包含2个sub-layer，分别是一个多头自注意力层和一个position-wise的前向神经网络，每个sub-layer都有一个residual connection以及一个layer normalization，即：

Osub−layer=LayerNorm(x+SubLayer(x))O_{sub-layer} = \text{LayerNorm}(x + \text{SubLayer}(x)) Osub−layer​=LayerNorm(x+SubLayer(x))

decoder: Nx=6N_x = 6Nx​=6，每一个layer中包含3个sub-layer，就是在encoder的2个sub-layer之间增加了1个多头注意力模型，该注意力模型以encoder的输出作为输入(query为上一个decoder output, key,value为encoder output)。同时修改了多头自注意力层，改为masked multi-head attention，以保证对position i的预测，只能依赖position i以前的内容

注意力机制(query, key, value)的本质，就是对value进行加权求和，其中权重来自f(query,key)f(query, key)f(query,key)。文章里用到的attention为：

Attention(Q,K,V)=softmax(QKTdk)V\text{Attention}(Q,K,V) = \text{softmax}(\frac{QK^T}{\sqrt{d_k}})V Attention(Q,K,V)=softmax(dk​​QKT​)V

multi-head attention
将Q,K,V通过不同的线性投影层，投影到其他维度的空间，最后将多个投影的结果拼接起来使用，例如2-head attention：
计算第一个Q,K,V：

Q1=W1QQK1=W1KKV1=W1VV\begin{aligned} Q_1 &= W_1^QQ \\ K_1 &= W_1^KK \\ V_1 &= W_1^VV \end{aligned}Q1​K1​V1​​=W1Q​Q=W1K​K=W1V​V​

类似地计算第二个Q,K,V：

Q2=W2QQK2=W2KKV2=W2VV\begin{aligned} Q_2 &= W_2^QQ \\ K_2 &= W_2^KK \\ V_2 &= W_2^VV \end{aligned}Q2​K2​V2​​=W2Q​Q=W2K​K=W2V​V​

拼接得到最终的Q,K,V：

Q=[Q1,Q2]K=[K1,K2]V=[V1,V2]\begin{aligned} Q &= [Q_1, Q_2] \\ K &= [K_1, K_2] \\ V &= [V_1, V_2] \end{aligned}QKV​=[Q1​,Q2​]=[K1​,K2​]=[V1​,V2​]​

position-wise feed-forward networks

FFN(x)=max⁡(0,xW1+b1)W2+b2\text{FFN}(x) = \max(0, xW_1 + b_1)W_2 + b_2 FFN(x)=max(0,xW1​+b1​)W2​+b2​

position-encoding
因为不是cnn/rnn类似的结构，所以为了让模型学到序列的顺序信息，加入了position encoding，用法为：

PE(pos,2i)=sin(pos100002idmodel)PE(pos,2i+1)=cos(pos100002idmodel)\begin{aligned} PE_{(pos, 2i)} &= sin(\frac{pos}{10000^{\frac{2i}{d_{model}}}}) \\ PE_{(pos, 2i+1)} &= cos(\frac{pos}{10000^{\frac{2i}{d_{model}}}}) \end{aligned}PE(pos,2i)​PE(pos,2i+1)​​=sin(10000dmodel​2i​pos​)=cos(10000dmodel​2i​pos​)​

其中：

  1. pospospos是position，iii是dimension

BERT: Pre-training of Deep Bidirectional Transformers for Language Understanding

发表于 2021-08-25 更新于 2021-08-26 分类于 技术 ， ml/dl ， 论文

终于来到大名鼎鼎的bert了

总结

随机mask掉sentence中的token，用多层、双向transformer进行pre-train，然后特定任务上再fine-tune

创新点

  1. 随机mask掉sentence中的token，以便使用双向transformer
  2. token representation = token embedding + segment embedding + position embedding

细节

分成2个阶段，pre-training和fine-tuning。在pre-training阶段，使用无标签的大量数据，在fine-tuning阶段，使用有标签的小量数据

bert的架构：multi-layer bidirectional Transformer encoder

使用了WordPiece作为embedding

pre-train阶段，使用无标签的数据：

对每个sentence pair，用SEP分开两个句子，然后对每个token增加一个embedding表示属于sentence A还是sentence B。
每个token的representation来自3个部分：token embedding, segment embedding, position embedding，即：

之前的LM都是单向的：要么从左向右，要么从右向左，理论上双向的模型应该比单向的模型效果好，但是双向的模型会直接“看到”要预测的目标。bert提出随机选择一部分token mask掉，然后预测这些mask掉的token

上面高亮的部分，正是ernie改进的点:ernie的mask不再是随机mask，而是选择句子中的entity进行mask

由于在fine-tune过程中，并没有MASK这个符号，所以在pre-train阶段，并不是将所有masked token都替换为MASK，而是：

  1. 80%替换为MASK
  2. 10%替换为随机token
  3. 10%保持不变

上面训LM的部分已经可以了，但是LM对句子之间的关系学习不好，因此还有专门对句子之间关系的模型。同样的训练数据，将sentence B改为：

  1. 50%情况下是sentence A的下一句，label = IsNext
  2. 50%情况下是语料库中的随机一句，label = NotNext

用上面图1的C作为判断next sentence的embedding

在多项任务上，输入可以是：

  1. sentence pairs
  2. hypothesis-premise pairs
  3. question-passage pairs
  4. text-∅\varnothing∅ pairs(for classification)

输出可以是：

  1. token级别的任务上(sentence tagging, question answering)，直接使用token representation
  2. 分类任务上，使用CLS representation

总结

在mask方式上做了改进，不再是从句子中随机mask掉token做“完形填空”，而是mask掉phrase/entity，让模型能学到更好的内容

细节

encoder部分：multi-layer transformer
对于给定的token，其embedding为将token embedding, segment embedding, position embedding求和得到

改变了mask方式，mask有2种方式：phrase-level mask, entity-level mask

basic-level mask
基础级别的mask，从句子中随机mask掉token
phrase-level mask
短语级别的mask，从句子中随机mask掉短语
entity-level mask
实体级别的mask，从句子中随机mask掉实体

ERNIE- Enhanced Language Representation with Informative Entities

发表于 2021-08-09 更新于 2021-08-26 分类于 技术 ， ml/dl ， 论文

才发现这篇不是百度的paper……尴尬了，搜了一圈发现，百度和清华同年都出了一篇paper，模型的命名都是ernie

总结

在传统的预训练语言模型中引入知识图谱，对token, entity分别用bert进行embedding，然后利用information fusion更新embedding，最终以token到entity的条件概率交叉熵作为损失函数

变量定义

token序列：{w1,…,wn}\{w_1, \dots, w_n\}{w1​,…,wn​}，其中nnn为token长度
实体序列：{e1,…,em}\{e_1, \dots, e_m\}{e1​,…,em​}，其中mmm为实体序列的长度
包含全部token的集合V\mathcal{V}V
包含kg全部实体的集合E\mathcal{E}E
如果一个token w∈Vw \in \mathcal{V}w∈V有对应的e∈Ee \in \mathcal{E}e∈E，则定义对应关系为f(w)=ef(w) = ef(w)=e

模型架构

T-encoder的layer为N
K-encoder的layer为M

对输入的token序列{w1,…,wn}\{w_1, \dots, w_n\}{w1​,…,wn​}及其对应的实体序列{e1,…,em}\{e_1, \dots, e_m\}{e1​,…,em​}，t-encoder先把token embedding, segment embedding, positional embedding加起来，得到input embedding，然后计算feature:

{w1,…,wn}=T-encoder({w1,…,wn})\{w_1, \dots, w_n\} = \text{T-encoder(}\{w_1, \dots, w_n\}\text{)} {w1​,…,wn​}=T-encoder({w1​,…,wn​})

其中：T-encoder是一个multi-layer的双向transformer（即bert）

用TransE模型把实体转化为实体embedding {e1,…,em}\{e_1, \dots, e_m\}{e1​,…,em​}，然后输入K-encoder计算final output embedding:

{w1o,…,wno},{e1o,…,emo}=K-encoder({w1,…,wn},{e1,…,em})\{w_1^o, \dots, w_n^o\},\{e_1^o, \dots, e_m^o\} = \text{K-encoder(}\{w_1, \dots, w_n\}, \{e_1, \dots, e_m\}\text{)} {w1o​,…,wno​},{e1o​,…,emo​}=K-encoder({w1​,…,wn​},{e1​,…,em​})

K-encoder结构

k-encoder由多层aggregator叠起来
对每第i层aggregator，其输入为：{w1i−1,…,wni−1},{e1i−1,…,emi−1}\{w_1^{i-1}, \dots, w_n^{i-1}\},\{e_1^{i-1}, \dots, e_m^{i - 1}\}{w1i−1​,…,wni−1​},{e1i−1​,…,emi−1​}

接下来对token和entity，分别经过multi-head self-attention MH-ATTs：

{w1i,…,wni}=MH-ATTs({w1i−1,…,wni−1}){e1i,…,eni}=MH-ATTs({e1i−1,…,eni−1})\{w_1^{i}, \dots, w_n^{i}\} = \text{MH-ATTs}(\{w_1^{i-1}, \dots, w_n^{i-1}\}) \\ \{e_1^{i}, \dots, e_n^{i}\} = \text{MH-ATTs}(\{e_1^{i-1}, \dots, e_n^{i-1}\}) {w1i​,…,wni​}=MH-ATTs({w1i−1​,…,wni−1​}){e1i​,…,eni​}=MH-ATTs({e1i−1​,…,eni−1​})

对有实体对应的token，计算information fusion layer:

hj=σ(W⃗tiwji+W⃗eieki+b⃗i)wji=σ(Wtihj+bti)eki=σ(Weihj+bei)\begin{aligned} h_j &= \sigma(\vec{W}_t^{i}w_j^i + \vec{W}_e^ie_k^i + \vec{b}^i) \\ w_j^i &= \sigma(W_t^ih_j + b_t^i) \\ e_k^i &= \sigma(W_e^ih_j + b_e^i) \end{aligned} hj​wji​eki​​=σ(Wti​wji​+Wei​eki​+bi)=σ(Wti​hj​+bti​)=σ(Wei​hj​+bei​)​

对没有实体对应的token，计算nformation fusion layer:

hj=σ(W⃗tiwji+b⃗i)wji=σ(Wtihj+bti)\begin{aligned} h_j &= \sigma(\vec{W}_t^{i}w_j^i + \vec{b}^i) \\ w_j^i &= \sigma(W_t^ih_j + b_t^i) \\ \end{aligned} hj​wji​​=σ(Wti​wji​+bi)=σ(Wti​hj​+bti​)​

其中：

  1. hjh_jhj​是内部的隐藏state
  2. σ(⋅)\sigma(\cdot)σ(⋅)是非线性激活函数，采用GELU函数

最后，根据每个token计算entity的概率：

p(ej∣wi)=exp(linear(wio)⋅ej)∑k=1mexp(linear(wio)⋅ek)p(e_j|w_i) = \frac{exp(\text{linear}(w_i^o) \cdot e_j)}{\sum_{k = 1}^m exp(\text{linear}(w_i^o) \cdot e_k)} p(ej​∣wi​)=∑k=1m​exp(linear(wio​)⋅ek​)exp(linear(wio​)⋅ej​)​

多机编辑blog

因为博客是在工作电脑上摸鱼摸出来的，周末回家有时候更需要在自己电脑上写字，所以就想在多个电脑上更新blog

网上搜到的一些解决方案，是把源码和生成的代码放在一个仓库的不同分支，但是我的博客代码库是开源的，源码想开私有库，所以我的方案是单独再开一个仓库用来保存源码。

新建的仓库用来放源码，把其中.deploy_git这个文件夹设为submodule即可。这个文件夹就是hexo部署生成的博客代码网址，也即github.io

加submodule的过程：

  1. 在原先代码上初始化仓库：git init
  2. 把博客代码文件夹加为submodule: `git submodule add <github.io url> .deploy_git
  3. 把其他代码推到仓库: git add .
  4. git commit -m 'init blog source'
  5. git add remote ...
  6. git push

在自己电脑上，git clone源码仓库，然后要先git submodule init

之后每次更新时，先pull远端内容，再push新的内容：

  1. 更新主仓库：git pull
  2. 将新的代码部署到submodule仓库中，hexo clean && hexo g -d
  3. 更新submodule仓库: git submodule update
  4. add, commit, push到主仓库
